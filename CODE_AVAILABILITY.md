# Code Availability

## Recommended Manuscript Wording

Use the repository URL once the public repository is live.

### Short code-availability sentence

`The code used in this study is available at https://github.com/1bytess/quantum-reservoir-battery-soh`

### Combined code and data availability statement

`The code used in this study is available at https://github.com/1bytess/quantum-reservoir-battery-soh. The public Warwick and Stanford datasets analyzed in this work are available from their original sources and should be obtained directly from those repositories. The ESCL laboratory dataset contains restricted data and cannot be publicly redistributed.`

## Practical Notes

- If the final GitHub repository name changes, update the URL in both the
  manuscript and this file.
- If you later archive a release on Zenodo, replace the GitHub-only sentence
  with the DOI-backed archive link.
- For review or early online release, GitHub is sufficient for the code link,
  but the data statement should remain explicit about public versus restricted
  datasets.
